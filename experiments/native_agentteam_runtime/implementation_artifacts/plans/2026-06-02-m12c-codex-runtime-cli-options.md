# M12c Codex Runtime CLI Options Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose the existing `CodexRuntimeAdapter` model, sandbox, and timeout controls through the public CLI so future role-specific agent runs can choose differentiated Codex configurations without changing code.

**Architecture:** Keep scheduler and adapter internals unchanged. Add CLI flags that are translated into `CodexRuntimeAdapter(...)` constructor arguments. Codex-specific options either run under `--runtime codex` or infer the Codex runtime; explicit fake/shell runtime choices reject Codex-only options.

**Tech Stack:** Python 3.12 standard library, `argparse`, `unittest`, `codex exec`.
